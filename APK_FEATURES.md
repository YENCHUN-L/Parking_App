# Parking App APK 現有功能說明

## 1. 應用定位
本 APK 為台北市停車場地圖 App，採用「內建資料優先」模式。
停車場資料從 App assets 讀取，並顯示在 OpenStreetMap 地圖上。

## 2. 資料來源策略
- 啟動時載入以下內建檔案：
  - `TCMSV_alldesc.json`
  - `TCMSV_allavailable.json`
- 不依賴執行期間即時下載停車場資料。
- 以停車場 ID 合併可用車位資訊（汽車/機車/大客車）。

## 3. 座標處理規則
- 座標優先順序：
  1. 入口座標 `EntranceCoord.EntrancecoordInfo[0]`
  2. `tw97x/tw97y` 轉 WGS84 作為 fallback
- 會做台灣範圍驗證（緯度與經度範圍）。
- 轉換後不在台灣範圍的資料會被略過，避免地圖視角被錯誤座標拉偏。

## 4. 地圖功能
- 使用 OSMDroid（MAPNIK）底圖，支援多點觸控。
- 初始縮放會限制在合理區間，避免太近或太遠看不到點位。
- Marker 採可視範圍全量渲染，拖曳/縮放後會即時重算。

## 5. Marker 顯示策略
- 採用 plain 藍色停車 pin（客製 Drawable）。
- 不設可視範圍內 marker 上限。
- 點擊 marker 後：
  - 開啟詳細資訊視窗
  - 地圖移動至該點

## 6. Marker 詳細資訊內容
目前資訊視窗包含：
- 名稱
- 行政區
- 地址
- 服務時間
- 汽車總格位
- 機車總格位
- 汽車可用格位
- 機車可用格位
- 座標來源（`entrance` 或 `tw97`）

## 7. 搜尋功能（地址/停車場）
- 地圖頁上方固定顯示搜尋面板。
- 搜尋流程：
  1. 先比對本地停車場 `name` 與 `address`
  2. 找不到時再用 Android `Geocoder` 補查
- 搜尋成功後：
  - 地圖跳轉到目標點
  - 縮放至少提升到 16
- 會在介面顯示成功/失敗訊息。

## 8. 目前位置功能
- 右下角浮動按鈕可回到目前位置。
- 未授權時會請求 `ACCESS_FINE_LOCATION`。
- 已取得定位時：
  - 立即移動到目前位置
  - 必要時把縮放拉到至少 16
- 首次定位尚未回來時：
  - 等 first-fix callback 再自動移動地圖。

## 9. 畫面狀態
- 載入中：顯示進度圈。
- 錯誤：顯示錯誤訊息。
- 無資料：顯示「沒有可顯示的停車場資料」。
- 頂部面板會顯示：
  - 目前可視範圍 marker 數量
  - 總載入停車場數量


## 11. 補充
- `Geocoder.getFromLocationName` 目前可用，但為官方標示 deprecated API。
- 本文件描述的是目前程式碼與現行 APK 的實作行為。
